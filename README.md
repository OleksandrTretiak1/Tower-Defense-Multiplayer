# Unity Tower Defense Game - Третяк Олександр

Цей проект представляє собою повноцінну стратегічну гру в жанрі Tower Defense, розроблену на рушії Unity. Основний акцент у роботі було зроблено на створенні комплексної ігрової логіки, що поєднує в собі економічну систему, складний штучний інтелект веж та багаторівневу структуру прогресії. Метою проекту було створення стабільного ігрового циклу з можливістю стратегічного вибору та збереженням особистих рекордів користувача.

## Ігрова механіка та бойова система
В основі ігрового процесу лежить система стратегічного розміщення оборонних споруд на ігровому полі. Кожна вежа функціонує на базі складних алгоритмів пошуку цілі, де пріоритет надається ворогу, який знаходиться найближче до кінця маршруту, що реалізовано через інтеграцію з системою вейпоінтів. У грі представлено два типи веж: стандартні кулеметні для швидкого точкового ураження та ракетні установки, що завдають шкоди по радіусу. Обидва типи споруд мають систему покращення до трьох рівнів, яка не лише змінює технічні характеристики (шкоду, дальність, скорострільність), а й візуальну модель об'єкта.

Економічна складова проекту базується на менеджері валюти, який контролює баланс гравця та можливість будівництва чи покращення веж. За кожного знищеного ворога гравець отримує винагороду, що дозволяє розвивати оборону динамічно під час бою. Система будівництва реалізована через контекстні меню (ShopUI та NodeUI), які з'являються безпосередньо над ігровими вузлами, забезпечуючи зручний User Experience та запобігаючи випадковим діям під час інтенсивних хвиль через перевірку на перекриття елементами інтерфейсу.

## Архітектура рівнів та система хвиль
Прогресія в грі побудована на системі хвиль, склад яких налаштовується через серіалізовані класи у спавнері. Це дозволяє створювати унікальні комбінації ворогів з різною швидкістю та рівнем здоров'я для кожного етапу. Кожен рівень містить перевірку на повне очищення карти від ворожих юнітів, і лише після знищення останнього ворога активується сценарій перемоги. Взаємодія з базою гравця реалізована через тригерну систему: якщо ворог досягає фінальної точки, він завдає шкоди загальному здоров'ю бази, що візуалізується через динамічну смужку здоров'я та звукові ефекти.

Важливою технічною особливістю є інтеграція системи збереження PlayerPrefs. Гра автоматично записує найкращий результат пройдених хвиль окремо для кожного рівня в реєстр операційної системи. Це забезпечує збереження прогресу навіть після повного закриття програми. В головному меню реалізовано динамічне зчитування цих даних, що дозволяє гравцеві бачити свої попередні досягнення безпосередньо під кнопками вибору рівнів.

## Інтерфейс та налаштування білду
Головне меню гри розроблено як багатофункціональний хаб, що містить основну панель керування, вікно вибору рівнів та систему налаштувань. Логіка навігації побудована на універсальному менеджері лобі, який керує станами панелей та переходами між сценами за назвами об'єктів. Окрему увагу приділено налаштуванням звуку: за допомогою UI Toggle реалізовано перемикач фонової музики, стан якого синхронізується з аудіо-системою та зберігається в пам'яті пристрою.

Для фінального білду під Windows було застосовано ряд технічних оптимізацій. Увімкнення вертикальної синхронізації (V-Sync) разом із обмеженням цільової частоти кадрів дозволило позбутися візуальних розривів та надмірного навантаження на систему. Також реалізовано функцію роботи у фоновому режимі, що дозволяє ігровому процесу продовжуватися навіть при згортанні вікна. Для зручності тестування та керування в меню додано функцію повного скидання ігрового прогресу, яка миттєво оновлює UI та очищує збережені рекорди.

## Технічні деталі керування
Весь функціонал гри повністю працездатний у фінальному білді. Керування в проекті здійснюється переважно за допомогою миші та частково клавіатури:
* Ліва кнопка миші - взаємодія з усіма елементами інтерфейсу, вибір точок будівництва (Nodes), купівля та покращення веж у меню.
* Клавіша Escape - виклик або закриття меню паузи під час ігрового процесу.
* Автоматизовані панелі (Win/GameOver) - надають доступ до кнопок перезапуску рівня чи повернення в головне меню за допомогою кліку.
* Перемикачі Music/Sounds - миттєве вимкнення або увімкнення звукового супроводу в головному меню та меню паузи.
* Кнопка Reset Progress - повне очищення збережених рекордів хвиль для всіх рівнів.

## Адаптація під мобільні платформи (Android)
Проект було модернізовано для забезпечення стабільної роботи на Android-пристроях з урахуванням особливостей мобільного геймплею:
* Touch Input: Систему керування адаптовано під тачскрін за допомогою `Pointer.current`, що забезпечує коректну взаємодію з об'єктами через дотики замість кліків миші.
* Safe Area Integration: Реалізовано динамічне підлаштування UI під вирізи та "чубчики" сучасних смартфонів за допомогою скрипта `SafeAreaHandler`, який коригує межі ігрового інтерфейсу.
* Dynamic Camera Scaling: Впроваджено систему автоматичного масштабування камери (`MapFitter`) для підтримки пристроїв з різним співвідношенням сторін (від вузьких смартфонів 21:9 до планшетів 4:3). Скрипт динамічно обчислює `orthographicSize` залежно від ширини екрана, що гарантує повне відображення ігрової карти без обрізання країв на будь-якому пристрої.
* Мобільна оптимізація: На ігрових рівнях розблоковано частоту кадрів до 60 FPS (`Application.targetFrameRate`) для максимальної плавності.
    * Увімкнено фізичну інтерполяцію для ракет, що усуває візуальне тремтіння об'єктів при русі.
    * Орієнтація екрана жорстко зафіксована в режимі Landscape для збереження цілісності інтерфейсу.
* Build Configuration: Фінальний мобільний білд зібрано у форматі `.apk` з оптимізованим стисненням текстур для зменшення розміру проекту та швидкого завантаження на пристрій.
